# 星阙 Horosa Windows 桌面版 `v1.0.9`

## 给谁下载

- **普通 Windows 10/11 x64 用户**：下载 `Horosa-Setup-1.0.9.exe`
- **如果你已经安装 `v1.0.8` 或更早版本**：建议升级到 `v1.0.9`
- GitHub Release 只保留离线安装器，不提供 portable zip 或源码快照式资产

## 三步安装

1. 下载 `Horosa-Setup-1.0.9.exe`
2. 双击安装器，按中文向导完成安装
3. 从桌面或开始菜单打开 `Horosa`（应用窗口标题仍显示“星阙”）

## 离线说明

- 安装包已内置 Electron、Java、Python、前后端资源
- 安装和本地功能可离线运行
- 普通用户机器不需要预装开发环境、浏览器、Java 或 Python
- 若要升级，请重新下载最新完整安装器覆盖安装
- 默认安装到 `%LocalAppData%\\Programs\\Horosa`
- 用户数据保存在 `%LocalAppData%\\HorosaDesktop`
- 升级、替换、修复默认保留用户数据

## 本次热修内容

### 1. 安装版本地排盘服务恢复

- 修复安装版内置 Python 图表服务与新版 `pyswisseph` 的 `azalt/cotrans` 兼容问题
- 现在普通星盘和相关排盘请求会正常返回，不再在启动后误报“本地排盘服务未开启 / 未就绪”
- 本机重装验收已确认：首窗出现、主界面加载、普通星盘排盘、关闭后重开和重复双击拉前台都恢复正常

### 2. 推运盘法达星限空白修复

- 修复 `法达星限` 在当前 chart 缺少 predictive 数据时整页空白的问题
- 现在进入 `法达星限` 时会按需静默补取法达数据，而不是要求用户先手工触发其它推运流程
- 页面会明确显示加载态、暂无数据态或加载失败态，不再出现“整页没内容但也没提示”的情况

### 3. 主限法与法达盘后天宫位标注恢复

- 修复 `主/界限法` 表格中丢失后天宫位与主宰宫位附加信息的问题
- 恢复 `(8th; 4R7R)` 这一类格式：前半段表示落入后天宫位，后半段表示主宰宫位
- `法达星限` 的主限/子限标签也重新遵循同一套后天宫位显示设置

### 4. 发布契约保持稳定

- 修复正常点击 `X` 关闭应用后误报 `Runtime error` / `Python chart service exited unexpectedly` 的问题
- 现在正常关闭只会留下 planned shutdown 日志，不会再把已计划的 Python/Java 退出记成故障
- Release 只提供 `Horosa-Setup-*.exe`
- 快捷方式继续为 `Horosa`
- 窗口标题继续显示“星阙”
- 不上传 portable zip、`latest.yml`、`.blockmap` 或其它自动更新元数据

## 适用对象

- **普通用户**：直接下载 `.exe` 完整离线安装器
- **开发者**：源码、脚本与桌面工程仍保留在 `main` 分支
